# CSVs in the EDCA Tool

## Purpose

- ALlowing user input
- Being super accessible and understandable format
- Everyone can use excel or googlesheets

## Formatting

- Do it right following the template

### Naming Conventions

- Required columns
- Numeric vs non/numeric data

### Mapping to Schemas

- Schemas (.yaml files) are linked
- Use them for the order and naming of each column
- That way pydantic will be easy

## Usage

- It's easy to edit just do it right

### Editing

- Using schemas as a baseline
- Copying the data in there
- Making sure everything follows preset units

### Inputting/Operating

- Make sure it maps to schemas
- Put in specific folder filepaths
- Make sure it's in there (and either commit it or leave it updated)
- Don't change column names or placement
